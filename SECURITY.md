# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in quoin, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

Instead, please send a report to the project maintainers by emailing:

- **security@quoin.dev** (preferred)

If email is not available, you may [report it as a GitHub Security Advisory](https://github.com/elcoosp/quoin/security/advisories/new) with limited visibility to maintainers.

### What to Include

Please include the following information in your report:

1. **Description** of the vulnerability and its potential impact.
2. **Steps to reproduce** the issue, including a minimal code example if possible.
3. **Affected versions** of quoin.
4. **Suggested fix** (optional, but appreciated).

### What to Expect

- We will acknowledge receipt of your report within **48 hours**.
- We will provide an initial assessment of the vulnerability within **7 days**.
- We will keep you informed of our progress toward a fix.
- If the vulnerability is accepted, we will coordinate a release timeline with you.
- We will credit you in the release notes (unless you request anonymity).

### Disclosure Policy

- We follow **coordinated disclosure**: we will not publicly disclose the vulnerability until a fix is available.
- If the vulnerability is already publicly known, we will prioritize the fix and may disclose details immediately.
- If we do not receive a response to follow-up questions within **90 days**, we may proceed with public disclosure.
